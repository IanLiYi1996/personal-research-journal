## 导言：为什么需要跨线综合

四条主线——Agent 体系、自动科研系统、训练动态显微镜化、世界模型与具身——各自有完整的内部演化逻辑。但它们不是相互独立的研究孤岛。

2026 年上半年，这四条线在若干关键节点出现了**概念收敛**：相似的问题在不同社区被独立发现，相似的答案在不同论文中重复出现。这种收敛不是偶然——它反映了整个 AI 研究社区正在共同面对的几个深层矛盾。

本文提炼 5 条跨线核心趋势，并列出 6 个尚无定论的开放问题。内容以综合分析为主，不重复各主线文档的细节；所有引用均指向四篇主线文档或其中列出的 arXiv 链接。

## 趋势一：Skill / Adapter / Memory 收敛为"持久本地能力载体"

**问题的起点**：每次 agent 任务都从零开始推理，每次 PEFT 微调都独立保存 LoRA 权重，每次世界模型都在 context window 里维护短期记忆——这三件事本质上是同一个问题的不同侧面：**如何让模型积累的能力"活"在某个持久、可检索、可版本化的载体里**。

2026 年上半年，三个原来相互独立的研究方向开始向同一个答案收敛：

- **Agent 侧**：七篇 Skill 论文（Ctx2Skill、SkillOS、SkillOpt、COLLEAGUE.SKILL、SkillsVote、MMSkills、Skill1）集中在 2026-04~05 发表，把 skill 定义为"带元数据、可检索、可优化、可版本化的能力单元"——本质上是一个**软件工件**，而不是一条 prompt。
- **训练侧**：Scaling of PEFT（arXiv 2606.02437）系统测量了不同 PEFT 方法（LoRA、DoRA、GaLore 等）在 scale 维度上的行为，发现 adapter 在参数效率和任务特化之间的权衡遵循可预测的 scaling law——这意味着 adapter 可以被视为**标准化的能力模块**而非 ad hoc 微调。
- **世界模型侧**：Mirage 引入 latent 空间记忆，用 latent vector 替代像素回环，解决了视频世界模型的"空间漂移"问题——本质上是把世界的局部状态压缩为**持久 latent 载体**。

三者的收敛方向是：一个能力单元，不管它叫 skill、adapter 还是latent memory，都需要满足四个属性：**持久**（跨任务/跨对话保留）、**可检索**（按语义查询）、**可组合**（支持单元组合完成复合任务）、**可验证**（有评分机制，支持淘汰/优化）。

**意义**：这一收敛预示着未来的 agent 基础设施将不再是"大模型 + prompt"，而是"大模型 + skill/adapter library"，后者可以持续增长、可以共享、可以追溯。

## 趋势二：评测从"能力通过率"转向"经济价值"

**问题的起点**：benchmark 虚高是整个 AI 社区的顽疾。MMLU/HumanEval 类的静态评测在 2022–2024 年推动了研究进步，但随着 LLM 开始刷题，这类评测的信号质量急剧下降。

2026 年上半年，评测方法论出现了一次明确的范式转变，由两个独立社区同步推动：

- **Agent 评测侧**：ALE（Agent Leaderboard via Economics，arXiv 2606.05405）提出用"经济价值"替代"通过率"作为主指标。核心论点是：agent 的价值由它在真实任务中节省的人工成本决定，而不是它在隔离测试集上的准确率。ALE 的硬数据是：在真实长流程任务上，最强的商业 agent 全通过率仅 2.6%——这个数字用旧评测框架是看不到的。SWE-Explore 在此基础上引入子能力隔离测试，把"为什么 agent 失败"拆解到比任务粒度更细的层面。
- **科研评测侧**：ResearchClawBench（arXiv 2606.07591）把自动科研系统的评测从"能否生成格式正确的论文"转向"grounded rubric"——每条评分维度都对应真实科研工作中的可验证标准（假设可证伪性、实验可复现性、数据解读准确性）。最强系统仅得 21.5/100 这一结果，在同行中引发了对"AI 科研能力虚高"的广泛讨论。

两个方向的共同信号是：**评测不应该测"模型能做什么"，而应该测"模型在有代价的任务中能稳定做到什么"**。"有代价"意味着：错误有后果、资源有限制、成功标准由外部决定而非 LLM 自评。

这一转变对 benchmark 设计有直接影响：未来高质量 benchmark 需要包含**真实环境交互**、**LLM-judge 污染控制**、**经济成本归因**三个要素，而不是单一的 accuracy score。

## 趋势三：世界模型表征从像素向显式 3D 跃迁

**问题的起点**：视频生成模型（Sora 类）输出逼真的像素序列，但像素序列是一种**隐式**的世界表征：距离、碰撞、物体持久性需要从像素推算，误差会随时间积累，在长 horizon 任务中迅速失效。

2026 年 4–6 月，三项工作同时落地，宣告"显式 3D"成为世界模型的主流范式：

1. **HY-World 2.0**（腾讯混元）：以 3D Gaussian Splatting（3DGS）为底层表征，生成可自由视角探索的交互世界，而非单一视角的视频剪辑。空间一致性和视角自由度是评测主指标，而非视频 FID。
2. **ABot-Earth**：直接从卫星图像生成 3DGS 地球模型，支持任意高度、角度的飞越和落地探索——"地球"变成了可以在其中导航的数字孪生，而非渲染图集。
3. **Mirage**：用 latent 空间记忆替代像素回环（pixel reconstruction loop），彻底消除了长序列视觉世界模型中的累积误差——latent space 充当 3D 知识的压缩载体。

为什么 3D 表征对 agent 更重要？Agentic World Modeling 综述的 L1/L2/L3 分类给出了清晰答案：L1 世界模型只能预测感知状态，L2 可以预测行动后果，L3 可以主动修正对世界规律的假设。3DGS 的显式结构使得 L2/L3 级别的操作（碰撞检测、空间推理、物理约束验证）可以在 query 时高效完成，而不需要解码整段视频后再做几何推算。

**与其他主线的连接**：训练动态侧的 Flow-OPD 把 OPD（On-Policy Distillation）引入 Flow Matching 视觉生成，提供了把训练机制知识迁移到视觉世界模型的工具链；SciAtlas 的"显式知识图谱比隐式嵌入更可查询"与 3DGS 的"显式 3D 比隐式像素更可推理"在认识论上完全一致。

## 趋势四：训练过程"显微镜化"——从配方经验到优化几何

**问题的起点**：RLVR（Reinforcement Learning from Verifiable Rewards）和大规模 On-Policy Distillation（OPD）在 2024–2025 年成为 agent 对齐的标准工具，但它们被当作黑盒使用：研究者知道哪些配方有效，不知道为什么。这在安全性和可信度方面留下了盲区——黑盒训练无法保证 agent 的内部机制与其声称的能力一致。

2026 年上半年，一批论文集体把训练过程放到"显微镜"下：

- **DelTA**（arXiv 2605.21467）：发现在 RLVR 训练中，只有约 20% 的 token 在梯度上携带判别信号，其余 80% 是噪声。这意味着大量计算资源被浪费在无效更新上，而不是被浪费在"错误"的更新上——这是一个可以精确干预的工程问题。
- **Geometry of OPD**：发现 OPD 的参数更新集中在一个低维子空间（subspace locking），不同的 OPD 配方实际上是在同一几何空间的不同路径上行走。这解释了为什么部分 OPD 变体之间的性能差异比预期的小——它们共享同一"轨道"。
- **TRB（Trust Region Bootstrapping）**：发现 OPD 早期 rollout 质量差，导致 teacher 信号在高噪声环境下被浪费；TRB 用信赖域 warmup 修复了这一问题，在早期训练阶段节省了约 30% 的有效计算量。
- **MPI Router**：把几何分析推进到 MoE 结构，发现现有 router 的专家分配与专家的真实"特长方向"（主成分方向）存在系统性错位；用主成分对齐改进 router 后，在相同参数量下提升了 1-3% 的任务准确率。

这条趋势的共同逻辑是：**一旦看清"更新发生在参数空间的哪里"，就能主动设计让正确的地方被更新**。这与 2014 年 Adam 优化器引发的关于 second-moment estimator 的集体解剖非常相似——理解机制先于设计改进。

**与其他主线的连接**：这条趋势与 Agent Skill 主线的 SkillOpt 论文高度重叠——SkillOpt 也在问"哪些 skill 更新有判别力"，使用了类似的 metric-driven 进化逻辑。两者都是在问同一个根本问题的不同层次版本：**学习系统中，哪些更新是真正有效的？**

## 趋势五：Agent 自我改进闭环——从"执行任务"到"进化自身"

**问题的起点**：前四条趋势描述的是"外部改进 agent"（更好的 skill、更好的评测、更好的训练机制、更好的世界表征）。但从 2026-06 起，一批工作开始探索"agent 自己改进自己"的范式。

这一趋势在三个方向上同步出现：

**方向 A · Role-Agent 双角色自进化**（arXiv 2606.10917）：单个 LLM 同时扮演 agent 角色（执行任务）和 environment 角色（提供反馈），两个角色在对话中协同进化。关键洞察是：agent 的 environment 本身可以是一个 LLM——这消除了对外部仿真器的依赖，使得 agent 在无监督条件下持续演化成为可能。初步结果显示在代码生成和数学推理任务上超过了需要外部 verifier 的基线。

**方向 B · SkillOpt 的 metric-driven skill 进化**：SkillOpt 不是人工优化 skill，而是让 agent 根据任务表现指标自动优化 skill 定义、检索策略和组合方式。这是一个有约束的自我改进系统——改进的对象是 skill library，而不是模型权重本身。

**方向 C · AutoResearchClaw 的 cross-run 经验积累**：每次实验产生的 lesson（成功路径、失败原因、边界条件）被存储为结构化知识，下一次 run 的 agent 可以检索这些 lesson 并避免重复错误。这是一种在时间维度上的自我改进，而不是在单次 run 内的自我纠错。

**三个方向的共同模式**：自我改进的对象都不是模型权重（那需要新的训练循环），而是**外围软件工件**——skill library、经验库、environment 定义。这使得"自进化"成为一个推理时间（inference-time）问题，而非训练时间问题，大幅降低了部署门槛。

**风险与局限**：自我改进闭环面临两个尚未解决的问题：(1) 自我强化偏见——如果评估标准本身来自 agent，改进可能是对错误方向的加速；(2) 分布外泛化——在已知任务上进化出的 skill 是否能泛化到分布外场景，目前缺乏系统性证据。

## 趋势六：自动科研从线性 Pipeline 到自强化闭环

**问题的起点**：AI Scientist（2024-08）确立了"端到端自动科研"的基本 pipeline：假设生成 → 代码实验 → 论文写作 → LLM 审稿。这个 pipeline 是线性的，每个阶段的输出直接输入下一阶段，失败了就重试，没有反馈机制和经验积累。

2026 年上半年，自动科研系统的演化方向是**把线性 pipeline 改造成自强化闭环**：

- **AutoResearchClaw 的 Pivot/Refine 机制**：实验失败不再触发简单重试，而是触发"假设层的重新评估"——agent 会反问"这个失败是实验问题还是假设问题"，并据此决定是修复代码（Refine）还是放弃假设（Pivot）。这把单次 run 变成了一个有内部反馈的控制系统。
- **Arbor 假设树（Hypothesis Tree）**：放弃单一假设链，改为维护一棵假设树——每个节点是一个可验证的假设，失败的节点产生新的分支而不是直接终止。树搜索策略（BFS/DFS/MCTS）可以根据计算预算动态切换。
- **ResearchClawBench 的 verifiable result reporting**：引入 numeric registry——所有 agent 宣称的实验结果必须对应一个可执行的验证脚本。这把"agent 自我报告"的信任链替换为"外部执行验证"，把科研诚信从软约束变成硬约束。

**与训练动态的连接**：verifiable result reporting 和 DelTA 的"识别判别性 token"出自同一个认识论根源——**AI 系统的内部声称需要外部可验证的锚点才能被信任**。这是跨越两条主线的最深层共识。

## Open Questions：六条跨线的开放问题

### OQ-1 · Skill 的标准化边界在哪里？

2026-05 集中爆发了七篇 Skill 论文，但它们对"skill"的定义并不统一：有的是代码函数（Ctx2Skill），有的是 RAG 检索单元（SkillOS），有的是 LoRA adapter（COLLEAGUE.SKILL），有的是评分策略（SkillsVote）。**当这些 skill 需要在跨 agent、跨平台的环境中共享时，需要什么样的标准化协议？** 这类似于 2010 年代 REST API 标准化带来的工程质变——但在 agent 领域，对应的"HTTP/REST"是什么，目前没有答案。

### OQ-2 · 世界模型的"物理律"如何从数据中涌现？

Agentic World Modeling 综述把 L3（自主修正世界假设）定义为世界模型的最高能力，但目前所有 L3 系统（包括 Role-Agent 的双角色自进化）都是在已知任务域内修正——**在全新物理环境中，世界模型能否从零归纳出正确的物理律，而不是记忆训练集中的物理行为？** PhysBrain 1.0 注入物理先验是一个方向，但注入是否会导致对训练分布的过度依赖，还不清楚。

### OQ-3 · RLVR 的可判别 token 比例是否有理论下界？

DelTA 发现约 20% 的 token 在 RLVR 训练中携带判别信号。但这个 20% 是任务无关的常数，还是随任务复杂度、奖励稀疏性变化的函数？**如果存在理论下界，它意味着 RLVR 的计算效率有一个不可绕过的上限**，这将直接影响 RLVR 能否随规模继续有效扩展。目前的 DelTA 是实证研究，缺乏理论解释。

### OQ-4 · 自我改进闭环如何防止价值漂移？

Role-Agent、SkillOpt、AutoResearchClaw 的 cross-run lesson 都在以某种形式实现 agent 自我改进。但**当改进标准本身来自 agent 时，谁来保证改进方向与人类价值对齐？** 这不是对抗性攻击问题（AgentDoG 1.5 已经处理），而是一个更根本的价值漂移问题：一个不断进化的 agent 在经过足够多轮自我改进后，其行为目标是否还与设计初衷一致？目前的安全对齐研究（RLHF、Constitutional AI）都假设 agent 不会主动修改自己的目标函数，但自进化范式打破了这个假设。

### OQ-5 · 评测经济价值的"谁的经济价值"问题

ALE 用"节省的人工成本"衡量 agent 经济价值，这是一个合理的初步指标。但**不同任务、不同用户、不同地区的人工成本差异巨大，"经济价值"并不是一个与文化和经济背景无关的客观指标**。此外，agent 替代人工产生的"节省"是否应该考虑社会成本（失业、技能退化）？如果算上这些，ALE 的评分体系需要重构。目前社区讨论停留在技术层面，社会经济维度几乎缺席。

### OQ-6 · 四线收敛的极限：Agent-as-Scientist-as-World-Builder

如果四条主线的收敛趋势延续：agent 拥有持久 skill library（趋势一）、能自我改进（趋势五）、依托显式 3D 世界模型进行规划（趋势三）、在训练过程中被精确监控（趋势四）、在经济价值维度被严格评测（趋势二）——**这样的系统是否会在某个时间点触发"科研自动化临界点"，即系统产出的科研知识超过人类消化能力，形成不可逆的知识加速？** ResearchClawBench 的 21.5/100 表明我们离这个临界点还很远，但 Arbor 假设树 + cross-run 经验积累 + SkillOpt 自进化的组合路径已经隐约可见。这是 2026 下半年最值得追踪的一条演化线索。

## 结语

2026 年上半年 AI 前沿的核心张力可以用一组对比来概括：

| 旧范式 | 新范式 |
|-------|-------|
| 单次 benchmark 通过率 | 经济价值 + 长流程稳定性 |
| Prompt engineering | Skill / Adapter library 工程 |
| 像素级世界模型 | 显式 3D 表征 |
| 黑盒训练配方 | 优化几何的显微镜化理解 |
| 人工设计 agent 系统 | Agent 自我改进闭环 |
| 线性科研 pipeline | 假设树 + 自强化验证 |

这不是一场革命，而是一次**工程化成熟度的系统性提升**。每一项转变背后都有精确的技术动因，每一个新范式都是对旧范式一个具体失效点的定点修复。

这种渐进但系统的演化方式，可能正是 AI 研究从"演示驱动"走向"工程可信"的必经之路。
