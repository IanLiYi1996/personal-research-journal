## 一、推理类型概述

### 1.1 主要推理类别

推理是以逻辑和系统的方式思考某事，利用证据和过去的经验得出结论或做出决定的过程。推理包括根据现有信息做出推论、评估论点并得出逻辑结论。

#### 演绎推理（Deductive Reasoning）

演绎推理是一种根据前提的真实性得出结论的推理类型。在演绎推理中，结论**必然**是从前提得出的，这意味着如果前提为真，则结论也必定为真。

```
前提：所有哺乳动物都有肾脏。
前提：所有鲸鱼都是哺乳动物。
结论：所有鲸鱼都有肾脏。
```

#### 归纳推理（Inductive Reasoning）

归纳推理是一种根据观察或证据得出结论的推理类型。根据现有证据，结论**可能**是正确的，但不一定是确定的。

```
观察：每当我们看到有翅膀的生物时，它都是鸟。
观察：我们看到一个有翅膀的生物。
结论：该生物很可能是一只鸟。
```

#### 溯因推理（Abductive Reasoning）

溯因推理是一种基于对给定观察集的最佳解释得出结论的推理类型。结论是基于现有证据**最有可能的解释**，但不一定是确定的。

```
观察：汽车无法启动，发动机下方有一滩液体。
结论：最可能的解释是汽车散热器泄漏。
```

#### 类比推理（Analogical Reasoning）

涉及对两个或多个事物进行比较，以便做出推论或得出结论。

#### 因果推理（Causal Reasoning）

涉及识别和理解事件或现象的原因和影响。

#### 概率推理（Probabilistic Reasoning）

涉及根据某些结果的可能性或概率做出决策或得出结论。

### 1.2 正式推理与非正式推理

- **正式推理（Formal Reasoning）**：是一个系统的逻辑过程，遵循一组规则和原则，通常用于数学和逻辑。更加结构化和可靠。
- **非正式推理（Informal Reasoning）**：是一种不太结构化的方法，依靠直觉、经验和常识来得出结论和解决问题，并且经常在日常生活中使用。更具适应性和开放性，但也可能不太可靠。

### 1.3 语言模型中的推理

语言模型中的推理概念已经存在一段时间了，但对其含义还没有明确的定义。在文献中，"推理"一词经常用来指非正式推理，尽管并不总是明确指出它是非正式的。根据任务、基准或所使用的方法，可以使用不同形式的推理，例如演绎推理。

## 二、公理化训练（Axiomatic Training）——训练大模型新范式

### 2.1 研究背景

因果推理（causal reasoning）是一种推理过程，遵守有特定因果性的预定义公理或规则。

#### Judea Pearl的因果关系阶梯

图灵奖得主Judea Pearl曾通过「因果关系阶梯」（ladder of causation）定义了可能的因果推理类型。

通常因果推理所用的公理或规则并不会被直接引入，模型学习的只是数据。公理或规则作为归纳偏差被纳入模型，比如通过正则化、模型架构或变量选择等方式。

#### 核心研究问题

这篇论文想要探讨的是：**模型能否从被动的符号演示中直接学习公理或规则**。作者将这种方法称为「公理化训练」（axiomatic training）。

#### 公理的表示形式

假设因果公理都可以以如下形式表示：**<前提，假设，结果>**，其中结果只有「是」和「否」两种形式。这基本类似于亚里士多德提出的「三段论」格式。

例如Judea Pearl书中提出的「碰撞公理」（collider axiom）：
```
前提：A ⊥⊥ B, B ⊥/⊥ C, A ⊥/⊥ C
假设：A是否导致C?
结论：是
```

#### 传递性公理的数学表达

论文引用了Judea Pearl和David Galles在1997年发表的研究，证明了对于给定的稳定概率因果模型，都存在一组有限公理，可以充分表征对应的有向因果图。

因果模型 M=(X,U,F) 被定义为内部变量X、外部变量U和一组结构方程F的集合，结构方程描述了变量X和U之间的因果关系。模型M的另一种等效表示方式是有向图G，用有向边 Vi->Vj 表示两个节点之间的因果关系。

所谓的「稳定概率」（stable probabilistic）因果模型，是指M中所有的不相关性(X ↛ Y|Z)都是稳定的。

在稳定性假设下，Galles和Pearl共描述了6个公理，而这篇论文主要关注**传递性公理**。对于稳定概率的因果模型，给定系统中的变量X、Y、Z，传递性公理可以写作：

> (X ↛ Y | Z) ∧ (Y ↛ Z | X) => (X ↛ Z | Y)

将上述表达式通过取反进一步简化，可以写出其含有因果相关性的版本：

> (X -> Z | Y) => (X -> Y | Z) ∨ (Y -> Z | X)

其中表达式左侧即为前提，右侧即为假设。这样的公理可以派生出数千个合成的符号表达式，从而用于向Transformer模型「教授」特定公理。

### 2.2 训练数据

上述含有前提和假设的公理能映射到「是」或「否」的标签，一条训练数据就可以表示为 **{(P,H,L)}** 的元组形式，其中：
- P = 前提（Premise）
- H = 假设（Hypothesis）
- L = 标签（Label），即「是」或「否」

给定一个真实的因果图，就可以通过应用传递性公理（一次或多次），枚举出所有可能的N个元组{(P,H,L)}，从而构建出数据集D。

**示例**：因果图中包含 X1->X2->X3->...->Xn 这样的链拓扑时：
- 一个可能的前提是 X1->X2 ∧ X2->X3
- 相应的假设 X1->X3 的标签为「是」
- 而另一个假设 X3->X1 标签就为「否」

**重要说明**：论文中为了表达的清晰性使用了数学语言进行描述，但**实际上用于训练的数据集只包含自然语言**。例如上面例子中的前提应该表达为「X1导致X2，且X2导致X3」。

### 2.3 数据扰动：泛化的关键

之前有研究表明，以「扰动」（perturbation）形式增加训练数据的可变性与多样性，有助于提升模型的泛化能力。因此，作者在不同层次上对训练数据引入结构化扰动，以最大化数据集分布的多样性。

#### 1）节点名称扰动

传递链上每个节点的名称都由1~3个字母/数字组成，长度和使用的特定字符是随机生成的。

#### 2）因果图拓扑结构扰动

主要包含两种类型：

- **顺序结构（sequential）**：所有的因果边方向都是从后向前，共同形成一个典型的「传递链」，比如 X->Y->Z 这种形式。
- **随机翻转（random flipping）**：给定一个顺序结构的传递链，对其中一些边进行随机翻转，从而引入复杂性。比如 X->Y->Z 可以被修改为 X->Y<-Z。

随机翻转可以在单一方向的链中添加：
- **分叉结构（fork）**：X<-Y->Z
- **碰撞结构（collider）**：X->Y<-Z

它们是任何有向因果图的基本构建块，有助于提升模型进行跨结构泛化的能力。

#### 3）链长度扰动

训练集中加入了长度不等的链，包含**3~6个节点**。

### 2.4 损失函数

论文**没有采用**训练Transformer模型常用的next token预测损失，而是根据给定数据集中每个元组的真实标签进行定义。损失函数直接优化模型对前提-假设对的分类能力（是/否判断），而非传统的自回归生成损失。

### 2.5 位置编码

除了训练数据和损失函数之外，另一个重要因素是位置编码的选择。之前有研究表明，位置编码机制对Transformer的序列长度泛化能力有明显影响，但不同的研究似乎得出了互相矛盾的结果。

因此，作者在研究中分别尝试了三种方法：
- **LPE（Learnable Positional Encoding）**：可学习位置编码
- **SPE（Sinusoidal Positional Encoding）**：正弦位置编码
- **NoPE（No Positional Encoding）**：无位置编码

训练和评估的整体流程：Transformer模型在顺序链和带有随机翻转的链上训练，长度为3~6个节点。之后，训练过的模型在具有>6个节点的更复杂结构上进行评估，其中节点平均的出度（out-degree）和入度（in-degree）都更大，序列更长，且引入了分支、反转（reversal）等复杂变化。

### 2.6 实现细节

- **架构**：基于GPT-2架构
- **参数规模**：6700万（67M）参数的解码器模型
- **模型结构**：12个注意力层、8个注意力头、512个嵌入维度
- **训练方式**：在各种训练数据集上从头开始训练
- **优化器**：AdamW，学习率 lr=1e-4
- **训练轮次**：100个epoch
- **位置编码**：分别实验了SPE、LPE和NoPE三种
- **自定义分词器（Custom Tokenizer）**：
  - 字母数字节点名称在**字符级别**进行分词
  - 像「causes」、「cause」、「Does」、「Yes」、「No」这样的特殊术语在**词级别**进行分词
  - 这种方法可以避免在测试时出现词汇表外（OOV）token
  - 最终**词汇表大小为69**

### 2.7 实验结果

#### 序列长度泛化（表1）

表1展示了不同模型在评估训练过程中未见过的更长因果链时的准确率。

- 在基线预训练语言模型中，**GPT-4在标准和随机翻转的因果链上都取得了最高的准确率**。
- 令人惊讶的是，尽管**TS2（NoPE）模型在训练过程中从未见过更长的序列**，但它的表现能够与万亿参数规模的GPT-4模型相媲美。
- 虽然训练时只用到了长度为3~6个节点的因果链，但序列长度为7~13时，**TS2（NoPE）在标准和随机翻转的链上，获得了比GPT-4更高或相当的准确率**。
- 对于序列长度为14-15的情况下，其准确率有所下降（标准链为0.85，随机翻转链为0.78），但仍然显著高于Gemini-Pro、Phi-3模型。
- 注意：随机预测会得到50%的准确率。

#### 节点名称转变（图3）

对于在TS2数据集上训练的模型，评估了其对变量名称变化的泛化能力：
- **TS2（NoPE）对节点名称的变化很稳健**，在引入新的、更长的名称时仍能保持较高的准确率。
- 它还保持了对新节点名称较长序列的通用性，其表现与GPT-4相似。

#### 因果序列顺序（表2）

与长度和节点名称的变化不同，反转（reversal）以及分支（branching）操作改变了因果结构，因此能更好地评估模型是否学习到了对结构的准确表示。

- **表2b**：TS2（NoPE）在长度不超过8的因果链上，获得的准确率高于Gemini Pro、Phi-3。长度为9时，TS2（NoPE）的准确率为0.73，与Gemini Pro（0.74）相当。
- **表2a**：对完全反转序列进行评估。公理训练模型TS2（NoPE）在限制链长度为3-6时，表现优于GPT-4。特别是，其准确率（长度为6的链为0.94）大大高于Gemini Pro和Phi-3（分别为0.62和0.69）。

#### 分支（表3）

分支可能是最有挑战性的任务，因为它引入了在训练期间未见的新结构。

- 虽然GPT-4在图大小不断增大的情况下获得了最佳准确率，但**TS2（NoPE）模型在除一个节点外的所有图大小上，都比Gemini Pro获得了更高的准确率**。
- 即使在有12个节点和1.4个分支因子的图形上进行评估，TS2（NoPE）模型也能获得**70%的准确率**，明显优于随机模型（50%）。

#### 总结

在所有评估设置中，公理化训练模型TS2（NoPE）的性能明显优于随机基线，即使因果链的长度超过其训练数据。特别是，模型没有在完全反转的链上进行训练，它的表现也与规模更大的GPT-4模型相当。在其他任务中，它的准确性往往优于或与Gemini Pro、Phi-3等十亿参数规模的模型相当。

**核心结论**：经过公理训练的模型可以从简单因果序列的演示中，学会推理更复杂的因果结构。这表明公理训练在因果图推理方面的潜力。

### 2.8 数据多样性和位置编码的作用

#### 位置编码的作用

比较不同位置编码选择的模型性能：

- **NoPE（无位置编码）**：在更长的序列（最长到15个节点的链）和复杂的、未见过的图结构上都能很好地泛化，尽管仅在3-6个节点的链上进行训练。
- **SPE（正弦位置编码）和LPE（可学习位置编码）**：在更长的链上表现也不错，但**当节点名称长度增加时表现较差**，即使是在节点数较少的链上也是如此（图3）。
- SPE和LPE的泛化失败，突出了模型无法处理训练集中序列的微小扰动。
- SPE在不同的结构维度上表现不佳（如分支）以及基于顺序的设置（shuffling和反转）。
- LPE在长度达9的线性链上表现良好，但之后急剧下降。

**结论**：研究结果扩展了早期关于NoPE有效性的研究，将其应用于理解因果序列的任务，并在测试时泛化到更长的长度和复杂的结构。

#### 数据扰动的重要性

- 仅在因果链上训练的模型可以泛化到较长的链（表1），但**不能泛化到其他DAG结构**（翻转、反转、分支）。
- 在TS1或TS2上训练的模型在所有情况下都具有通用性，包括随机翻转、顺序排列和分支——这突出了通过随机翻转在边水平上纳入可变性的影响。
- 不过，研究发现**TS2的准确率高于TS1**，即使TS1因随机翻转而产生了更多变化。
- **关键发现**：虽然扰动有助于结构泛化，但**过度的扰动可能会阻碍结构泛化**。

### 2.9 从相关性推断因果关系（图5）

研究人员进一步探索了公理化训练的能力迁移，将其应用于从观察数据中的相关性陈述推断因果关系的任务。

**任务描述**：每个数据实例包括用自然语言描述的3到6个节点图的相关关系；目标是推断假设的真值，判断任何给定节点之间是否存在直接或间接关系，以及可能存在的碰撞节点和混杂因素。这个任务比应用传递性公理要困难得多。

**实验结果**：

| 模型 | 准确率 |
|------|--------|
| Gemini Pro | ~52%（接近随机猜测） |
| Phi-3 | ~52%（接近随机猜测） |
| GPT-4 | 58% |
| **公理化训练的小型Transformer** | **64%** |

- 预训练模型（Gemini Pro、Phi-3）的表现与随机猜测相似（准确率为52%）
- GPT-4表现稍好，但性能仍然较低（准确率为58%）
- **小型Transformer模型表现优于所有基线模型，准确率达到64%，比GPT-4高出6%**

总的来说，研究人员认为公理化训练是教Transformer模型学习因果关系的一种很有前景的方法。通过进一步探索不同的训练设置，公理化训练的Transformer模型可能会在这类因果推理任务上得到进一步的优化。
