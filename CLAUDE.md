## What This Is

A personal research journal: markdown notes organized into folders, browsable via Docsify, managed by a shell CLI.

## Commands

```bash
npm install                          # Install docsify-cli
npm run docs                         # Start Docsify at http://localhost:3000

./journal.sh paper <name>            # Create papers/YYYY-name.md from template
./journal.sh note <name>             # Create research-notes/YYYY-MM-DD-name.md from template
./journal.sh weekly                  # Create weekly/YYYY-WXX.md for current week
./journal.sh search <keyword>        # Grep across all markdown files
./journal.sh index                   # Regenerate README.md indexes in each folder
./journal.sh serve                   # Alias for npm run docs
```

## Architecture

- **Docsify** serves the repo root as a static site (configured in `index.html`). No build step — it reads `.md` files directly. Sidebar navigation is in `_sidebar.md`. The search plugin provides full-text search in-browser.
- **`journal.sh`** is a bash CLI that creates notes from templates, runs search, and regenerates folder indexes. It always `cd`s to the script's own directory so it works from anywhere.
- **Folder README.md files** are auto-generated by `./journal.sh index`. Don't hand-edit them; they get overwritten. The index command reads the `# heading` from each note to build the link list.

## Paper Figures Convention

Each paper's images go in a subfolder matching the note filename (without `.md`):

```
papers/2025-agentic-rl-survey.md
papers/2025-agentic-rl-survey/
  fig1-survey-structure.jpg
  fig2-paradigm-shift.jpg
```

Reference images from the markdown with relative paths: `![caption](2025-agentic-rl-survey/fig1.jpg)`.

## Templates

- `papers/_template.md` — paper notes (Authors, Venue, TL;DR, Method, Results, Strengths/Weaknesses, Key Takeaways)
- `research-notes/_template.md` — research notes (Date, Tags, Context, Main Content, Open Questions, References)

When creating notes, always copy from the template rather than writing from scratch.

## HF Daily Papers Digest Workflow

This is a recurring task — writing weekly digests of Hugging Face Daily Papers.

### Workflow

1. **Fetch papers**: Call HF API `GET /api/daily_papers?date=YYYY-MM-DD&limit=50&sort=publishedAt` for each date
2. **Deduplicate**: Cross-check against the previous digest to exclude already-covered papers (match by arXiv ID)
3. **Deep dive selection**: Pick 1-2 papers with the highest impact (upvotes + novelty + technical depth), fetch full paper via `https://huggingface.co/papers/{ID}.md`
4. **Download figures**: From `https://arxiv.org/html/{ID}v1/x{N}.png`, save to `research-notes/{note-name}/` subfolder
5. **Write digest**: Chinese, grouped by themes, include overview table → per-paper summaries → deep dives → trend analysis → open questions
6. **Update indexes**: Run `bash journal.sh index`

### Digest Format

- **File**: `research-notes/YYYY-MM-DD-hf-daily-papers-{date-range}.md`
- **Figures subfolder**: `research-notes/YYYY-MM-DD-hf-daily-papers-{date-range}/`
- **Language**: Chinese (中文), with English terms for technical concepts
- **Structure**: Context → 论文总览表 → 分主题详解 → 深入分析(1-2篇) → 趋势分析 → Open Questions → References

## Docsify Plugins

The site uses these plugins (configured in `index.html`):

- **KaTeX** (`docsify-katex@1` + `katex@0.16`) — renders `$...$` inline and `$$...$$` display math
- **Mermaid** — diagram rendering
- **Search** — full-text search
- **Image Zoom** — click to enlarge images
- **Flexible Alerts** — callout blocks
- **Tabs** — tabbed content
- **Copy Code** — code block copy button
- **Pagination** — prev/next navigation

## Things to Know

- `_sidebar.md` controls Docsify navigation. Update it when adding new top-level sections.
- `.nojekyll` is required by Docsify — don't delete it.
- After creating or deleting notes, run `bash journal.sh index` to keep folder indexes current (note: use `bash journal.sh`, not `./journal.sh` which may lack execute permission).
- The `resources/README.md` is hand-maintained (not auto-generated by the index command).
- `wechat-drafts/` is excluded from Docsify navigation — for WeChat article drafts only.
